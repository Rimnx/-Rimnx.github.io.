HTML

  • !Doctype is the webpage.
  • lang="en" is the language it'll show up as aka english.
  • head the top of the webpage invisable
  • meta invisible data
  • charset chart made back in the day for comuters to recognize letters and alphabets, even emojis now
  • title the name of the tab of the webpage
  • body the bulk of the webpage also invisable
  • h1 - h6 headings 1 being the biggest and 6 being the smallest
  • p for paragraph or also text can be used for any writting
  • ul un ordered list
  • ol ordered list aka 1. 2. 3.
  • li is for list that have bullet points
  • img is for images
  • a and href are for links
  • footer is the bottom section of the webpage invisable as well
  • sections are to organize the pages content
  • div is for dividers
  • br is for page break or "white space"
  • HTML stands for hypertext markup language

CSS

  • div is a selector and groups the values and properties
  • property is for example the color in a color: blue
  • value is in the same example above the blue
  • the group of proerty and value is called a declaration
  • commas let us seperate multiple elements so one piece of code can affect multiple elements allowing us not to retype the same code
  • css stands for cascading style sheet

Git

  • pwd stands for print working directory aka check where in the file in at
  • ls list of files in directoy
  • cd change directory
  • cd .. go back one step
  • mkdir make new directory
  • touch name.txt creates file in text editor
  • git status checks what branch im in
  • git add -a add changes
  • git commit -m"x" add message to new commit
  • git checkout main takes me to the main branch
  • git checkout -b feature/add-html creates a new branch called html

Javascript

  • variables can be changed for example letmyvarible="bob" can be written as myvariable="joe" later on
  • strings are a group of letters aka words or phrases and they always have quotation marks surrounding them as in the example above with joe and bob.
  • numbers don't have quotes around them and can be any number within the gigabytes range if bigger then the system cannot calculate every number
  • boolean is true or false
  • arrays allow us to store data in little "boxes" to help us quickly identify them later
  • objects can be anything
  • to comment in JavaScript we use /*x */ x being the text inside
  • addition and subtraction are represented the same as on paper + and -
  • Multiplication and division are represented as * and /
  • to assign something any value, property, etc we use an = sign
  • truthy is how we describe === sign it means the truth to a degree or close enough to being true
  • similarly not truthy is represented as !, or !==
  • a function is usually looks like a variable followed by ()
  • anonoumous functions are followed by ()=> instead of just ()
